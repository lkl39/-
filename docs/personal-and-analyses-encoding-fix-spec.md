# 个人页面与分析记录乱码修复规范

## 背景

- `public/inner-pages/个人页面/code.html` 与 `public/inner-pages/分析记录/code.html` 当前存在静态文案乱码。
- 问题根因是文件内容本体已经被错误编码覆盖，而不是浏览器未按 `utf-8` 解析。
- 这两个页面仍承担现有静态 inner-page 展示职责，需要先恢复可读文本和基础导航。

## 目标

- 恢复两个页面的中文文案为正常可读状态。
- 保留现有静态页面结构、样式与动态脚本逻辑。
- 不修改后端接口契约，不扩散到其它 inner-page 页面。

## 范围

- `public/inner-pages/个人页面/code.html`
- `public/inner-pages/分析记录/code.html`

## 方案

### 恢复基线

- `个人页面` 以历史提交 `47ce2dc` 中的正常中文版本为恢复基线。
- `分析记录` 以历史提交 `f8b8cc7` 中的正常中文版本为恢复基线。

### 最小补丁

- 对恢复后的文件补齐当前仍需要的帮助中心与技术文档链接。
- 不重写页面结构，不重做 UI。
- 保留现有 `fetch("/api/inner-data?...")`、退出登录、跳转等脚本逻辑。

## 验证标准

- 页面标题、导航、表单标签、按钮文案恢复正常中文。
- 页面内不再出现明显的 `鏅鸿兘`、`涓汉` 一类乱码片段。
- `帮助中心` 与 `技术文档` 链接不再互相指错。
