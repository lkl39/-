# Fake Entry Cleanup Spec

## 背景
- 规则页头部的“新增规则”和右下角 AI 魔法棒目前没有真实后端能力支撑。
- 规则页“过滤条件”是演示按钮，但页面内部已经存在真实筛选区。
- 个人页“最近登录设备”当前没有设备审计数据来源，只是占位提示。

## 目标
1. 保留并接通真正可用的筛选入口。
2. 删除没有真实动作支撑的规则页假入口。
3. 删除没有数据源支撑的个人页安全占位区。

## 实施范围
- `components/dashboard/pages/rules/rules-page.tsx`
- `components/dashboard/pages/account/account-page.tsx`

## 决策
- “过滤条件”：保留，接到页面内真实筛选区。
- “新增规则”：删除，等待后端创建能力明确后再恢复。
- AI 魔法棒悬浮按钮：删除，避免误导用户。
- “最近登录设备”：删除，等待设备审计数据接入后再恢复。

## 不做项
- 不新增规则创建接口或弹窗。
- 不新增登录设备审计能力。
- 不调整其它页面结构。

## 验收标准
1. 规则页“过滤条件”按钮能定位到真实筛选区。
2. 规则页不再显示“新增规则”和 AI 魔法棒按钮。
3. 个人页不再显示“最近登录设备”区块。
4. `eslint` 与 `npm run build` 通过。
