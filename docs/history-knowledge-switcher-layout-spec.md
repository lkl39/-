# 历史与知识三页切换条布局规范

## 背景
- 历史问题库页面中的三页切换条当前位于运营卡片与筛选区之间。
- 用户进入页面后，首先看到的是数据卡片，切换入口被挤到中部，难以快速判断当前所在页面与相邻页面关系。

## 目标
- 将“三页切换条”提升为页面级导航，而不是内容区组件。
- 让用户在进入页面首屏时就能明确看到：
  - 当前所在页
  - 另外两个相邻页面
  - 三页之间的关系属于同一工作流

## 布局要求
1. 切换条必须放在页面标题说明之后、任何业务卡片之前。
2. 历史问题库页中的切换条不再放在中部内容区，也不再作为筛选区的一部分出现。
3. 当前页必须有明显的视觉高亮，包括：
   - 更高对比度的边框与背景
   - “当前”字样或等价状态提示
4. 非当前页保持可点击，但视觉权重低于当前页。

## 范围
- 仅调整历史问题库页面布局。
- 不改动规则层、模型层、RAG 层和数据结构。
