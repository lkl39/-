# Sidebar Upload Entry Correction Spec

## 背景
- 当前侧栏“日志分析”被改成直接进入分析记录，和用户期望不一致。
- 上传页侧栏底部按钮仍显示“开始新分析”，语义重复。

## 目标
1. 让左侧菜单“日志分析”恢复进入上传日志页。
2. 让上传页侧栏底部按钮改为“进入分析记录”，并跳转到分析记录页。
3. 保持现有高亮分组逻辑，不影响其它菜单。

## 实施范围
- `components/dashboard/shell/dashboard-sidebar.tsx`

## 不做项
- 不修改上传页内容区。
- 不修改人工复核页与分析报告页。
- 不调整接口、字段与路由结构。

## 验收标准
1. 左侧菜单“日志分析”点击进入 `/upload`。
2. 侧栏底部按钮文案为“进入分析记录”，点击进入 `/dashboard/high-risk`。
3. 上传、分析记录和详情页仍归属于同一菜单高亮组。
4. `eslint` 与 `npm run build` 通过。
