# 帮助中心与技术文档页面规范

## 目标

- 将侧边栏中的 `帮助中心`、`技术文档` 从占位入口接成真实站内页面。
- 保持现有静态内页结构不变，不引入新的复杂路由或文档系统。
- 先提供可扩展的目录页，覆盖当前系统最常见的使用场景与说明主题。

## 范围

- 新增 `public/inner-pages/帮助中心/code.html`
- 新增 `public/inner-pages/技术文档/code.html`
- 统一修改现有静态内页中底部的 `帮助中心`、`技术文档` 入口链接

## 不做的内容

- 不新建数据库表
- 不接外部帮助系统
- 不实现全文检索
- 不重写现有页面导航结构

## 页面内容

### 帮助中心

- 页面定位：面向最终使用者的操作帮助页
- 内容模块：
  - 快速开始
  - 日志上传与分析
  - 分析报告怎么看
  - 人工复核怎么处理
  - 历史与知识怎么用
  - 常见问题

### 技术文档

- 页面定位：面向内部、交付或演示场景的系统说明页
- 内容模块：
  - 系统架构概览
  - 规则层、模型层、RAG 层职责
  - 主要数据流
  - 导出与复核说明
  - 当前系统边界

## 交互约束

- 采用纯静态展示，不额外依赖接口
- 保留与现有内页一致的侧边栏与顶栏风格
- 入口统一使用相对路径：
  - `../帮助中心/code.html`
  - `../技术文档/code.html`

## 兼容要求

- 不修改现有字段与接口
- 不影响已有静态页的主功能脚本
- 只替换帮助入口的 `href="#"`，不动其他占位锚点
