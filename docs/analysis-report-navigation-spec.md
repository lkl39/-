# 分析报告导航链路规范

## 目标

- 上传分析完成后，默认进入“分析报告”页，而不是历史日志存档或其他中转页。
- 历史日志存档、分析记录中的“查看报告/查看详情”必须可点击，并跳到同一个报告入口。
- 分析报告页按 `logId` 读取真实数据，不展示固定假数据。

## 统一入口

- 报告页统一入口为 `/dashboard/analyses?logId=<id>`
- 静态 iframe 内部可继续使用 `../分析报告/code.html?logId=<id>` 作为等价入口

## 数据约束

- `logs` 时间字段统一使用 `uploaded_at`
- `analysis_results`、`log_errors` 保持使用各自的 `created_at`

## 兼容要求

- 不改分析引擎与写库逻辑
- 不重做历史日志存档和分析记录页面，只修正数据读取与报告跳转
