# 个人页面右上角头像同步规范

## 背景

- 个人页面当前会更新中间的大头像预览。
- 右上角导航栏中的小头像仍使用静态初始地址，保存资料后不会同步变化。
- 这会让用户误以为头像修改未生效。

## 目标

- 保持现有个人页面结构与接口契约不变。
- 保存资料或输入头像 URL 后，右上角小头像与大头像预览同步更新。
- 对持久化头像地址增加版本参数，避免浏览器缓存导致“已保存但没变”。

## 范围

- `public/inner-pages/个人页面/code.html`

## 方案

- 给右上角导航头像增加可定位的 DOM 标识。
- 抽出统一的头像更新函数，同时更新：
  - 页面主头像预览
  - 右上角导航小头像
- 读取资料时，使用 `updatedAt` 组装带版本号的头像地址。
- 保存资料成功后，使用接口返回的 `avatarUrl` 与 `updatedAt` 立即刷新两个头像。

## 验证标准

- 输入头像 URL 时，页面主头像和右上角小头像同时变化。
- 保存资料后，刷新前即可看到右上角头像变化。
- 再次打开个人页面时，两个头像仍保持一致。
