# 静态历史页编码恢复规范

## 背景
- `public/inner-pages/历史日志存档/code.html` 与 `public/inner-pages/分析记录/code.html` 当前存在整页文字乱码。
- 乱码不仅影响显示，还把部分侧边栏与内部跳转链接改坏，导致点击后进入不存在的页面并出现白屏。
- 这两个页面仍是 dashboard iframe 模式下的真实入口，因此需要先恢复静态 HTML 编码与链接稳定性。

## 目标
- 仅恢复受损静态页的文本与链接。
- 保持原有页面布局、配色与交互结构不变。
- 保留当前真实报告跳转：`/dashboard/analyses?logId=<id>`。

## 方案
1. 使用 `HEAD` 中未损坏的文件版本恢复两个静态页。
2. 在恢复后的文件上，重新应用真实报告查看链接。
3. 不改上传分析流程，不改其他 dashboard 页面。

## 边界
- 不重做页面样式。
- 不改数据库接口字段。
- 不改 `日志上传` 与 `分析报告` 页面结构。
