# 问题中心建议趋势与复核效率真实化改动说明

## 1. 改动目标
- 将“系统处理建议趋势”“今日复核效率”区块由静态假数据改为真实数据，且保持原有视觉样式不变。

## 2. 后端接口调整
- 文件：app/api/inner-data/route.ts
- view=incidents 返回值新增：
  - suggestionTrend：近7天每日建议数量（数组，含 label/count/heightPercent）
  - reviewEfficiency：今日已复核/总待复核数及百分比
- 保持 rows 字段兼容，前端无需改动表格渲染逻辑。

## 3. 前端页面调整
- 文件：public/inner-pages/问题中心/code.html
- “系统处理建议趋势”区块：
  - 静态描述改为柱状图，动态渲染 suggestionTrend 数据
- “今日复核效率”区块：
  - 百分比、进度条、已复核/待复核数均由 reviewEfficiency 动态渲染
- 其余样式与布局保持不变。

## 4. 字段兼容性
- 未影响原有 rows 字段及表格分页逻辑
- 新增字段仅供前端挂点使用

## 5. 依赖与约束
- 依照 AGENTS.md 及 field-spec.md 规范，所有新增字段与聚合逻辑均不破坏现有契约
- 未修改规则检测、RAG 检索、模型输出等主链路

## 6. 验证方式
- 启动后端，访问“问题中心”页面，确认“系统处理建议趋势”“今日复核效率”区块数据为真实聚合结果，且样式无变化。

---

如需回滚，仅需恢复 route.ts 和 code.html 对应区块为原静态实现。
