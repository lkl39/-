# 性能分析演示回退规范

## 背景
- `/dashboard/performance` 已迁移为真实 Next 页面。
- 当前页面在 Supabase 未配置、用户未登录、或窗口期真实样本过少时，会返回全 0 数据，导致三模式对比、指标卡和模式明细接近空白。

## 目标
1. 保持现有 `PerformancePageData` 字段主体兼容，不重写页面结构。
2. 真实数据可用时优先展示真实聚合结果。
3. 真实数据不可用或不足以支撑三模式对比时，回退到可展示的演示数据。
4. 页面必须明确标记当前数据来源，避免把演示数据误认为生产结果。

## 回退策略
1. 以下场景直接使用演示数据：
   - Supabase 环境变量未配置。
   - 当前用户未登录。
2. 以下场景视为“真实样本不足”，使用演示数据：
   - 当前窗口期总任务数为 0。
   - 当前窗口期总分析结果数为 0。
   - 当前窗口期只有 0 个或 1 个模式存在有效任务量，无法形成三模式对比。
3. 真实数据满足对比条件时，继续返回真实聚合结果。

## 字段约束
1. 保留现有 `PerformancePageData` 字段结构。
2. 允许新增兼容字段用于标识数据来源：
   - `dataSource.kind`: `real` 或 `demo`
   - `dataSource.label`: 页面展示文案
   - `dataSource.description`: 说明文案

## UI 要求
1. 页面顶部范围标识区展示当前数据来源。
2. 演示数据状态下，推荐说明与洞察文案必须指向“演示口径”而非“真实运行窗口”。
3. 不新增复杂交互，不修改原有导出按钮和范围切换方式。

## 接口要求
1. `/dashboard/performance` 首屏数据与 `/api/inner-data?view=performance` 的返回口径必须复用同一份构造逻辑。
2. 不再维护两份独立的 performance 聚合实现，避免首屏与切换结果不一致。

## 不做项
1. 不改规则层、模型层、RAG 层字段。
2. 不新增数据库表或持久化演示数据。
3. 不在本轮实现真实三模式重跑任务。
