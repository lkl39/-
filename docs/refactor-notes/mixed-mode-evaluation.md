# 模式效果对比页面改造记录

## 本次目标

- 将 `性能分析` 页面从“实验展示页”收敛为“模式效果对比页”。
- 用当前窗口期真实运行数据，说明混合模式为什么更适合作为默认分析方案。

## 本次改动

- 为 `view=performance` 新增 `focusMetrics` 与 `recommendation` 输出。
- 保留原有 `metrics / chart / modes / insights` 字段兼容，不重写原聚合逻辑。
- 将页面标题从 `算法性能对比实验` 调整为 `模式效果对比`。
- 新增顶部推荐区，直接给出“默认推荐：混合模式”的结论和证据。
- 三张指标卡调整为：
  - 混合模式准确性
  - 混合模式覆盖率
  - 混合模式平均延迟
- 表格区标题从 `实验详细指标` 调整为 `模式明细`。

## 资源控制

- 页面继续复用当前窗口期 `logs` 与 `analysis_results` 聚合。
- 页面访问不触发三模式实时重跑。
- 本次没有引入新的 benchmark 扫描任务。

## 后续可选项

- 如果后续要做更强证据页，可单独增加“固定验证集评估”模块。
- 当前版本先满足产品表达：证明混合模式更好用，而不是构建严格离线实验平台。
